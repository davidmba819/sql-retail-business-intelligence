# Retail Business Intelligence & Customer Analytics System

## Project Overview

This project focuses on building a SQL-driven Retail Business Intelligence & Customer Analytics System using a multi-vendor e-commerce marketplace dataset. The goal is to transform raw transactional data into structured business insights that support customer analysis, revenue tracking, seller performance evaluation, operational monitoring, and customer experience analysis.

The project is designed as an end-to-end SQL analytics portfolio project with emphasis on relational data modeling, analytical querying, reporting logic, and business intelligence workflows.

---

## Business Problem

Modern e-commerce marketplaces generate large volumes of transactional and operational data across customers, sellers, products, payments, logistics, and reviews. Without a structured analytics system, it becomes difficult to monitor business performance, identify operational bottlenecks, understand customer behavior, and support data-driven decision-making.

This project addresses these challenges by building a structured SQL-based analytical environment for marketplace intelligence and reporting.

---

## Project Objectives

The main objective of this project is to build a scalable SQL analytics system capable of:

* Analyzing customer purchasing behavior
* Monitoring revenue and sales trends
* Evaluating seller performance
* Exploring product and category performance
* Tracking logistics and delivery efficiency
* Understanding payment behavior
* Measuring customer satisfaction through review analysis

---

## Dataset Overview

The dataset represents a multi-vendor e-commerce marketplace containing information related to:

* Customers
* Orders
* Order items
* Products
* Sellers
* Payments
* Reviews
* Product categories
* Geolocation data

The database structure follows a relational design where transactional and descriptive tables are connected through primary and foreign key relationships.
